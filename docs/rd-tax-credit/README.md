# 研發投資抵減：文件管理與知識庫

> 適用範圍：公司向政府申請研發投資抵減相關之計畫、申請、佐證與查核文件，以及內部知識庫（FAQ、SOP、範本）。

---

## 一、建議分類架構總覽

依「申請生命週期 + 政府/查核需求」設計，便於歸檔、檢索與佐證：

```text
docs/rd-tax-credit/

│   ├── index.md                 # 研發抵減文件治理入口
│   └── retention-and-audit.md   # 保存年限與查核對應（可選）
├── plans/                       # 計畫書與申請主體文件
│   ├── index.md                 # 年度計畫總覽
│   └── 2025/                    # 依年度分目錄
│       └── 2025_年度研發投資抵減計畫書.md
├── application/                 # 申請與送件
│   ├── index.md                 # 申請流程說明
│   └── checklist-YYYY.md       # 當年度送件檢核表（可選）
├── execution/                   # 執行紀錄與佐證
│   ├── index.md                 # 佐證分類說明
│   ├── manpower/                # 人力投入（工時/比例佐證）
│   ├── expenditure/             # 經費分攤與憑證對應

│   ├── index.md                 # 查核準備說明
│   └── YYYY/                    # 依年度或查核案分

│   ├── index.md                 # 範本清單與使用說明
│   └── 年度研發投資抵減計畫書（政府計畫格式）.md
```

## 二、分類說明與用途

| 分類 | 目錄 | 用途 | 典型文件 |
|------|------|------|----------|
| **計畫書** | `plans/` | 當年度研發計畫之主體文件，對應政府計畫格式 | 年度研發投資抵減計畫書、計畫變更說明 |
| **申請送件** | `application/` | 送件流程、檢核表、補件紀錄 | 送件檢核表、補件清單、收件證明 |
| **執行佐證** | `execution/` | 人力、經費、成果之佐證與對應關係 | 工時表、經費分攤表、研發成果報告/程式/規格 |
| **查核備查** | `audit/` | 國稅局/主管機關查核時之準備與回覆 | 查核通知、回覆說明、意見紀錄 |
| **範本** | `templates/` | 與政府格式一致之空白/範例，避免每年重寫 | 政府計畫格式範本、檢核表範本 |
| **知識庫** | `kb/` | 內部 FAQ、釋疑、年度申請 SOP | 資格認定、費用認列、查核常見問題、申請步驟 |

## 三、與政府計畫書章節的對應

您提供的計畫書（政府格式）章節可與本分類對應如下，方便日後佐證時對號入座：

| 計畫書章節 | 建議存放／佐證位置 |
|------------|--------------------|
| 一、申請單位基本資料 | `plans/YYYY/` 計畫書內；變更時可於 `application/` 留紀錄 |
| 二、研發計畫名稱 | `plans/YYYY/` |
| 三、研發計畫期間 | `plans/YYYY/` |
| 四、研發背景與動機 | `plans/YYYY/` |
| 五、研發目標 | `plans/YYYY/` |
| 六、研發內容與技術方法 | `plans/YYYY/`；細部佐證可放 `execution/deliverables/` |
| 七、研發人力配置與投入比例 | 佐證 → `execution/manpower/`（工時、比例、名單） |
| 八、研發經費概況 | 佐證 → `execution/expenditure/`（分攤表、憑證對應） |
| 九、研發成果與效益 | 佐證 → `execution/deliverables/`（報告、產出物） |
| 十、結語 | `plans/YYYY/` |

## 四、知識庫（KB）建議主題

- **資格與適用**：哪些活動可列研發、產業別認定、計畫範圍界定。
- **費用認列**：薪資分攤、設備使用、委外、可認列與不可認列項目。
- **人力佐證**：投入比例計算方式、工時紀錄要求、名單與職務對應。
- **查核準備**：常見查核重點、書面/實地查核注意事項、回覆範例。
- **年度申請 SOP**：從計畫撰寫、內部審查、送件到補件的步驟與負責人。

## 五、與現有文件治理的關係

- 本專區為**公司層級**（研發抵減申請），非單一產品模組，故獨立於 `docs/modules/` 之外，置於 `docs/rd-tax-credit/`。
- 文件格式與撰寫仍建議遵循 [Markdown 合規規範](../templates/Markdown_合規規範.md)，並可依需要引用 `governance/` 之 metadata 與 AI 檢索設定。
- 若需納入 AI 客服或內部檢索，可在 [Metadata Routing](../governance/metadata-routing.md) 中為 `rd-tax-credit` 設定路由與權限。

## 六、資料檢核與還缺項目

送件或查核前可依 **[還缺哪些資料](./還缺哪些資料.md)** 逐項檢核，該文件彙整：
- 目前已具備的資料清單
- 仍缺或待補項目（計畫書欄位、人力／經費／成果佐證、申請檢核表、KB、查核準備）
- 建議優先補齊順序與送件前勾選表
