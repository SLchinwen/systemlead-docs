## 一、文件定位與用途

- **文件性質**：正式專案預算報價報告（可歸檔、可長期維護）
- **適用對象**：【客戶名稱】、系統整合商、採購單位、資訊單位
- **文件用途**：
  1. 說明本案屬於 **e首發票加值中心標準服務 + 客製外掛專案**
  2. 明確區分「年度標準服務費」與「一次性專案開發費」
  3. 作為內部預算編列、採購審查與專案核准依據

> 本案所有服務與系統功能，均建置於 **e首發票加值中心雲端平台（Microsoft Azure）**，
> 非部署於【客戶名稱】或系統整合商自管主機環境。

---

## 二、e首發票加值中心服務說明（標準服務）

### 2.1 服務定位

- e首發票為依法設立之 **電子發票加值服務中心**
- 採 **多客戶共用之雲端集中式架構**，由矽聯科技負責維運
- 核心服務包含：
  - 銷項／進項電子發票開立與管理
  - 財政部平台介接、上傳與申報
  - 查詢、報表與 API 整合能力

### 2.2 雲端與資安特性

- 平台：Microsoft Azure
- 架構特性：多層式服務架構、排程上傳與異常重送機制
- 治理與稽核：權限控管與操作紀錄、資料拋轉歷程可追蹤、符合電子發票加值中心營運與資安規範

## 業務填寫指引與計算原則

> 填寫報價書前，請依本節逐項確認。**資料不足時，務必詢問客戶補齊**，不得自行假設金額。

### 查表項目（對應 [線上報價產品引導](./online-quoting-product-guide.md)）

| 項次 | 填寫位置 | 查表來源 | 說明 |
| ---- | -------- | -------- | ---- |
| 1 | 三、3.1 銷項級距與年費 | Step 21 年度張數級距 | 依客戶選定之方案（訂單版/發票版）與級距，查對應年費。**若客戶未提供預估用量，需詢問：貴司年度預估開立張數？** |
| 2 | 三、3.2 輔導協辦 | Step 11 | 客戶無字軌時才列。訂單版/發票版/機台：約 3,600 未稅；境外版：約 30,000。**以系統或公司公告為準**。 |
| 3 | 三、3.2 系統開通 | Step 12 | 訂單版 3,600、發票版 21,000、境外版 36,000、機台 12,600（未稅）。**以系統或公司公告為準**。 |
| 4 | 三、3.3 進項年費 | 進項模組定價 | 通常約 6,000/年。**若無公告，需詢問產品/財務**。 |
| 5 | 三、3.4 選購模組 | Step 30～46 | 蝦皮 API、雲列印、B2B 等，依客戶需求查對應年費/一次性。**若客戶需求不明，需詢問：需要哪些加購模組？** |

### 計算原則

| 項次 | 計算方式 | 注意事項 |
| ---- | -------- | -------- |
| 六、工時小計 | 各 P 項工時相加 | 工時含分析設計、開發、測試、文件，不得僅估「寫程式」時間 |
| 六、金額小計 | 工時 × 單價 | 單價依公司標準（例：1,200/人時），**不得自行調整** |
| 三、銷項小計 | 年費（首年）+ 輔導協辦（如需）+ 系統開通 | 輔導協辦僅在客戶無字軌時列入 |
| 七、總覽 | 專案開發費 + 銷項標準服務 + 進項年費 + 選購 | 確保與各節數字一致 |

### 資料不足時需詢問客戶的項目

| 類別 | 詢問項目 | 目的 |
| ---- | -------- | ---- |
| **用量與方案** | 年度預估開立張數？訂單版或發票版？混合使用？ | 決定銷項級距與報價方案（混合用一律發票版） |
| **字軌與開通** | 是否已有電子發票字軌？是否為首次建立該統編帳號？ | 決定是否列輔導協辦、開通費金額 |
| **選購模組** | 是否需要蝦皮 API、雲列印、B2B、進項模組等？ | 決定三、3.4 與 3.3 項目與金額 |
| **客製需求** | 具體要哪些 API、哪些欄位、哪些情境？ | 決定五、功能範圍與六、工時拆解 |
| **單價與折扣** | 是否有合作專案、特殊折扣？ | 確認是否可適用優惠（須符合折扣治理） |

- [ ] 六：工時已拆解為可管理模組，每項皆有工時與小計，合計 = 工時 × 單價

- [ ] 資料來源與取號規則已確認（訂單版/發票版/混合），混合一律發票版報價
