## e首發票服務與報價引導說明（客服與客戶用）

> **文件性質：非 SSOT（說明文件）**  
> 本文件用於客服/業務在協助客戶「線上首購下單」時，能一致地解釋 **服務差異、費用項目、選用時機、年度張數規則與優惠折扣**。具體價格與活動以系統與公司公告為準。

---

## 1. 使用情境與前提

- 本文件適用於 **e首發票線上首購**（引導表單下單流程）。
- 客戶會先選擇使用方式（訂單版/發票版/境外版/機台專用），再依序選擇：
  1) 是否需要「協辦申請資格與字軌」
  2) 系統開通與線上教育訓練（一次性）
  3) 首購折扣（不定期）
  4) 年度張數級距（主約/年費）
  5) 需要的選購模組與儲值服務

> 注意：**金額為負數代表折扣**（例如 -3,150 即折抵 3,150 元）。

---

## 2. 方案選擇總覽（第一頁）

客戶先選擇「發票來源/使用方式」。產品代碼與顯示名稱以 [pricing-tiers.yaml](./pricing-tiers.yaml) 為準，表單與系統請使用同一組 **id**。

### 2.1 訂單版（plan_order）

- **顯示名稱**：訂單版
- **以「訂單」為核心**：客戶提供訂單資料，e首發票負責取號與開立並回傳發票號
- **取號責任**：加值中心集中取號，較不會有重複取號問題
- 常見客群：電商、網站接單、訂單 EXCEL 匯入、系統 API 串接、需訂單查詢與追蹤

### 2.2 發票版（plan_invoice）

- **顯示名稱**：發票版
- **以「已取號發票」為核心**：客戶端已取號之發票上傳至加值中心，負責記錄、上傳、管理與申報
- **取號責任**：客戶或前端系統取號；加值中心需處理重號、傳送失敗重傳、逾期與稽核問題，驗證與配套 SOP 較多，**故費用較訂單版高**
- 常見客群：既有 ERP/收銀系統已取號，需上傳、查詢、管理

### 2.2.1 資料來源與取號規則（報價時引用）

報價與對客戶說明時，可引用 [定價治理辦法 - 資料來源與取號規則](./pricing-governance.md#05-報價前必區分資料來源與取號規則)：

- **混合使用**：營業人若同時使用訂單版與發票版，**一律以發票版取號報價**，避免取號責任與稽核爭議。

### 2.3 境外版（plan_overseas）

- **顯示名稱**：境外版
- **境外電商版（限境外電商）**
- 常見客群：境外電商 C 類發票需求

### 2.4 機台專用（plan_kiosk）

- **顯示名稱**：機台專用
- **手持機台、繳款機台**等設備開立之專屬使用服務
- 常見客群：販賣機、無人繳費/繳款設備、現場機台交易

## 3. 首購下單流程（步驟對照）

> 依表單步驟代號，整理「會出現哪些費用」與「何時該選」。

### Step 11｜（選購）輔導協辦申請電子發票資格與字軌（一次性）

**選用時機：**

- 客戶尚未取得電子發票資格或尚未申請字軌
- 希望由 e首發票協助準備申請流程、降低錯誤

**不需選用：**

- 已有電子發票字軌
- 客戶要自行向國稅局申請

**費用項目（依方案不同）：**

- 訂單版/發票版/機台專用：特惠 3,780（含稅顯示）
- 境外版：特惠 30,000

> 備註：此為「開通與輔導一次性費用」，非年費。

---

### Step 12｜系統開通＋線上教育訓練（一次性開通費，建立統編帳號）

**選用時機：**

- 首次建立該統編之 e首發票帳號與平台使用權限
- 會包含線上教育訓練/導入說明（依方案）

**費用項目（依方案）：**

- 訂單版：3,780
- 發票版：21,000
- 境外版：36,000
- 機台版：12,600

---

### Step 13｜首購優惠/合作專案折扣（不定期，金額為負數）

**選用時機：**

- 僅限符合「線上首購」或「合作專案」活動條件
- 客服應提醒：折扣項目依活動期間可能變動

**常見折扣類型：**

- 訂單版：首購限時折抵、首購＋推薦優惠
- 發票版：首購限時折扣、合作店家折扣（如萬州通）
- 境外版：首購折扣（-1000 / -2000）
- 機台專用：設備專案折扣（販賣機/頂尖設備）

> 備註：折扣通常限定「線上申辦與付款」。

---

### Step 21｜年度張數級距（主約/年費，決定當年度可開立張數）

**選用時機：**

- 客戶必選（除特定期繳方案另列）
- 依當年度預估開立量輸入**張數**（或千張），由系統依級距公式算出年費

**計價方式（非單價×數量）：**

- **輸入**：年度張數，以**千張**為單位（例：1 = 1,000 張、240 = 240,000 張）
- **輸出**：年費（元）。採**分段線性內插**，相鄰錨點間線性計算，體積愈大單張邊際愈低
- **錨點與公式**：見 [級距公式與維護說明](./pricing-tier-formula.md)，錨點資料維護於 [pricing-tiers.yaml](./pricing-tiers.yaml)
- **範例（訂單版）**：300 張 → 2,000 元；1,000 張 → 3,000 元；240,000 張 → 20,000 元（實際以 YAML 錨點為準）

**重要規則（客服需主動說明）：**

1. **年度張數使用期限：**
   - 一般方案使用到「當年度 12/31」為止
2. **年中開通之彈性：**
   - 部分方案可提供「免費延展」：例如年中後開通，可建議購買較高級距，延展使用到次年底（依活動/方案規則）
3. **未用完張數的付費延展：**
   - 當年度剩餘張數 **超過 1/2** 時，可申請 **付費延展一次**（依公司規則/客服公告）
4. **張數不足的加購：**
   - 可加購張數補足
   - 加購通常 **不適用首購/預購張數優惠**（依活動）

**各方案級距錨點來源：**

- 訂單版、發票版等之錨點（千張 → 年費）均定義於 [pricing-tiers.yaml](./pricing-tiers.yaml) 之 `tierAnchors`；境外版、機台專用若採「一期×6」或不同級距，可於同檔擴充或另檔維護。
- 系統實作：依 YAML 與 [級距公式](./pricing-tier-formula.md) 計算費用，無需在程式內寫死級距表。

> 提醒：Step 21 多數為「首購特惠價」；若非首購或活動結束，價格/級距可能改為一般年費。

## 4. 選購模組與儲值服務（加購項目）

> 客戶完成「主約年度張數」後，可依需求加購模組。以下依用途分類，方便客服引導。

### 4.1 訊息通知儲值（Step 22）

**用途：** 簡訊通知、LINE 通知、異動通知、或其他客製通知服務扣點

**項目：**

- 儲值金 60 點 / 300 點 / 1200 點

**扣點規則（摘要）：**

- 簡訊：約 3 元/則
- LINE：約 0.25 元/則
- 其他服務依實際扣點
- 未使用完：解約結算以折讓方式退款

---

### 4.2 中獎發票領取/代列印儲值（Step 23）

**適用情境：**

- 有開立 C 類發票（或客戶希望提升中獎通知/列印便利）

**項目：**

- 多元領取（通知＋超商列印）：10 張 / 100 張
- 代列印（店家交付或郵寄給中獎人）：10 張 / 100 張

**治理備註：**

- 儲值額度可延續至下一期使用
- 解約：扣 100 元手續費返還

---

### 4.3 訂單/電商整合與轉檔（Step 30～32、47～49）

**適用情境：** 電商平台、訂單來源多、需自動開立/統計/品名處理

- EC 匯入線上轉檔授權（一次性）：EC 訂單轉檔一格式授權（特價）
- 蝦皮 API 開立（單店/多店）：含 API 開通與年度使用費（依方案）
- MO 店 API 開立：年度使用費（依方案）
- 蝦皮訂單折扣統計開立發票（一次性開通）
- 蝦皮訂單置換品名作業（年度）
- 訂單批次作廢當期發票（一次性開通）

> 客服提醒：多店方案通常需先具備對應的基礎整合（依備註規則）。

---

### 4.4 雲 POS / 雲列印 / 設備與列印服務（Step 33～35、38）

**適用情境：** 現場開立、需要列印、需要 POS 操作

- 雲 POS 年繳（體驗優惠）：11 個月 / 23 個月
- 雲列印 IOT 開通授權（一次性）
- 雲端列印服務費（年繳）：2026 或 2026+2027
- 熱感印表機/設備方案：
  - 自備主機＋EPSON
  - Star MCP3
  - 樹梅派＋印表機（若開放）
  - Aclas 手持 POS（6 吋/8 吋，含設定與開通）
- 雲端錢櫃開啟服務（by 台）：需配合雲列印與設備

---

### 4.5 FTP / SFTP 傳輸服務（Step 36）

**適用情境：** EXCEL 匯入量大、需以檔案傳輸自動化

- FTP 傳輸年度服務費（通常配合主約續約選購）

---

### 4.6 機台雲發票/雲列印（Step 37）

**適用情境：** 無人販賣機、繳款機台等「每台」計費服務

- 機台專屬 ID 串接基本＋雲發票（每台年費）
- 機台專屬 ID 串接基本＋雲列印（每台年費）

---

### 4.7 歷史資料存查與後台管理（Step 38、41）

- 歷史發票資料線上存查續約（可查 5 年內）
  - 注意：若年度中斷未續購，重啟需另繳「資料回復費 5,000」
- 增加後台管理帳號（全年度續約）

---

### 4.8 指定日期開立/媒體申報/通知/交換模組/客戶資料管理（Step 42～46）

- 指定開立發票日期服務：需留切本；逾期需異動單並另收處理費
- 進銷項媒體申報管理模組：年度使用費（整合電子與非電子發票、產生標準申報檔）
- 通知 MAIL 相關服務：通知信發送、內容客製或送達追蹤（年度）
- B2B 交換模組：銷項/進項/全模組（年度）
- 客戶資料管理：增加 EMAIL CC 通知、設定不上傳財稅平台等（年度）

## 5. 客服引導話術（建議）

### 5.1 先確認使用方式

- 「請問您發票是由訂單產生（訂單版），還是 ERP 已取號（發票版）？」

### 5.2 再確認是否需要字軌協辦

- 「如果您已經有字軌或公司會自行申請，就不需要選協辦；沒有的話我們可以協助一次性輔導。」

### 5.3 年度張數選擇重點

- 「年度張數用到當年底；若您是年中後才開通，我們會建議選較適合的級距，部分方案可延展到次年底。」
- 「若年底剩餘張數超過一半，通常可付費延展一次（依規則），避免浪費。」

### 5.4 加購模組引導

- 「您是否有電商平台（蝦皮/MO/其他）要自動開立？或現場要用 POS 與列印？」
- 「如果有開立 C 類，我會建議您預購中獎通知/超商列印或簡訊服務，會比較順。」

---

## 6. 治理提醒（內部一致性）

- 金額為負數：一律視為折扣，需說明「活動期間與適用條件」。
- 首購優惠：通常限定「線上申辦與付款」；若客戶改人工流程，可能不適用。
- 年度張數延展/加購：屬政策性規則，客服應依最新公告/規則說明，避免承諾超出標準。
- 硬體/設備：需確認客戶是否具備網路環境與可執行 IOT 程式之設備（筆電/桌機/樹梅派等）。
