## 業務報價 SOP

> **文件性質：非 SSOT（說明文件）**  
> 本 SOP 為業務報價流程之內部作業指引。具體核准權限與系統操作以公司公告為準。

---

## 一、SOP 目的

本 SOP 說明 **e首發票標準服務（年度張數 + 選購模組）之報價流程**，  
協助業務與客服在報價時步驟一致、檢核完整，避免遺漏或錯誤承諾。

> **適用**：標準方案報價  
> **不適用**：專案／混合報價 → 依 [專案報價治理辦法](./project-quoting-governance.md) 與 [報價預算書範本](./project-quote-budget-template.md)

---

## 二、報價流程

```
需求確認 → 方案選擇 → 報價準備 → 檢核 → 核准（如需）→ 送出
```

### Step 1：需求確認

- 確認客戶類型：新客 / 續約 / 加購
- 確認用量：歷史張數或預估月均/年度
- 確認需求模組：電商整合、POS、雲列印、通知、中獎、B2B 等
- 確認有無特殊需求 → 若有，判斷是否需走專案報價

### Step 2：方案選擇

- 依用量選定年度張數級距（參考 [業務報價 QA Q11](./sales-quoting-qa.md#q11客戶用量不確定該建議哪個級距)）
- 勾選所需選購模組
- 若有折扣需求，確認是否符合現行折扣政策及核准權限

### Step 3：報價準備

- 使用公司指定報價工具/表單
- 填入：客戶名稱、方案級距、模組、價格、有效期限
- 若有折扣：填入折扣條件、核准人、期間

### Step 4：檢核

- 依 [報價檢核清單](#三報價檢核清單) 逐項確認

### Step 5：核准（如需）

- 依公司規定：哪些情境需主管/財務核准
- 取得核准後再送出

### Step 6：送出

- 以公司指定方式送出報價（Email / 系統 / 正式報價單）
- 留存報價紀錄，便於後續追蹤與續約

---

## 三、報價檢核清單

送出報價前，請確認：

| 檢核項 | 說明 |
| ------ | ---- |
| 客戶資訊正確 | 公司名稱、統編、聯絡人 |
| 方案級距正確 | 符合客戶用量建議，未過度或不足 |
| 模組清單完整 | 客戶需求之模組皆有列入 |
| 價格與有效期限 | 價格正確、有效期清楚標示 |
| 折扣合規 | 若有折扣，符合政策且已取得核准 |
| 無口頭承諾 | 未承諾超出報價單之內容 |
| 標準說法一致 | 對客戶說明符合 [定價治理辦法 - 對外說明](./pricing-governance.md#4-對外定價說明的標準結構) |

## 四、情境判斷

### 4.1 標準報價 vs 專案報價

| 情境 | 走標準 SOP | 走專案報價 |
| ---- | ---------- | ---------- |
| 年度張數 + 選購模組 | ✓ | |
| 首購 / 續約 / 加購 | ✓ | |
| 需客製開發、特殊整合 | | ✓（見 [專案報價治理辦法](./project-quoting-governance.md)） |
| 非標準 SLA、專屬維運 | | ✓ |
| 多產品捆綁、特殊條款 | | ✓ |

### 4.2 需主管/核准之情境

依公司規定，常見包含：

- 折扣超過 X%
- 特殊付款條件
- 非標準合約條款
- 首年免費/大幅優惠

> 具體核准層級與條件，以公司公告為準。

---

## 五、錯誤與風險提醒

- **勿口頭承諾價格**：一切以正式報價單為準
- **勿擅自給折扣**：須符合折扣政策並取得核准
- **勿混淆標準與專案**：專案需求應轉交專案流程，勿用標準報價涵蓋
- **勿忽略檢核清單**：遺漏項目可能導致後續爭議或合規問題
