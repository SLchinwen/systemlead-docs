# 發票異動判斷 SOP
##（作廢 vs 折讓）

## 一、SOP 目的

本 SOP 用於判斷 **當電子發票已開立後，若發生交易取消、內容錯誤或金額調整時，應選擇「作廢」或「折讓」處理**，避免使用錯誤流程造成帳務或申報風險。

> 本文件為「情境判斷 SOP」，不說明實際操作步驟。  
> 實際畫面與操作方式，請依對應操作手冊執行。

---

## 二、適用情境

- 客戶取消交易
- 發票金額、品項或數量開立錯誤
- 部分退貨或事後折扣
- 客服需快速回覆「該作廢還是折讓」

---

## 三、判斷流程（核心）

### Step 1：該張發票是否「尚未完成申報 / 上傳」？

- **是** → 優先考慮「作廢」
- **否** → 進入 Step 2

> 是否可作廢，需以系統與法規允許狀態為準。

---

### Step 2：是否為「整筆交易取消」？

- **是** → 使用「作廢」
- **否（僅部分錯誤或調整）** → 進入 Step 3

---

### Step 3：是否僅需調整「部分品項或金額」？

- **是** → 使用「折讓」
- **否** → 回到作廢流程或重新評估交易情境

---

## 四、常見情境快速判斷表

| 情境 | 建議處理方式 |
| --- | --- |
| 客戶整筆退貨 | 作廢 |
| 僅退回部分商品 | 折讓 |
| 金額誤開（部分） | 折讓 |
| 完全未成交 | 作廢 |
| 發票已跨期 | 折讓（較常見） |

---

## 五、錯誤使用風險提醒

- 已申報完成之發票，通常 **不可直接作廢**
- 不得以折讓方式處理「完全不存在的交易」
- 作廢與折讓一旦完成，皆不可修改，只能再做後續處理
