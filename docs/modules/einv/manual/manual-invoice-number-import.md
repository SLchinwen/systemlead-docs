# 手動匯入字軌號碼操作手冊

## 一、手冊目的

本手冊說明 **e首發票（EINV）系統中「手動匯入字軌號碼」的實際操作方式**，適用於系統未能自動取得財政部字軌資料時，由營業人自行下載並匯入字軌檔案，以確保後續發票可正常開立。

> 本文件為 **非 SSOT 操作說明文件**，僅說明操作流程。  
> 字軌來源、取號責任與治理規則，請以規則文件為準。

---

## 二、適用情境

- 當期字軌號碼已用完，需緊急新增本數
- 系統尚未自動匯入下期字軌
- 營業人未授權矽聯為加值中心，系統無法自動取得字軌資料

---

## 三、字軌來源說明

發票字軌號碼需由營業人自行登入 **財政部電子發票整合服務平台** 下載。

- 檔案格式：CSV
- 檔名通常以 `invnoapply` 開頭
- 內容包含：
  - 發票期別
  - 字軌代碼
  - 起號 / 迄號

---

## 四、操作步驟

### （一）進入字軌號碼管理

後台路徑：

```
電子發票管理系統
＞ 發票字軌與銷項申報管理
＞ 發票字軌號碼管理
```

---

### （二）匯入 CSV 字軌檔案

1. 點選【匯入 CSV】按鈕
2. 選擇以 `invnoapply` 開頭的 CSV 字軌檔案
3. 點選【上傳】
4. 系統顯示匯入成功訊息

> 匯入成功後，系統將自動完成字軌切本配號。

---

### （三）匯入後確認

請務必確認字軌已可正常使用：

1. 仍於【發票字軌號碼管理】頁面
2. 確認新匯入字軌：
   - 期別正確
   - 字軌號碼範圍正確
   - 狀態為可用

確認完成後，即可進行發票開立作業。

---

## 五、注意事項

- 匯入檔案 **必須為 CSV 格式**
- 不建議使用 Excel 開啟後另存，避免格式遭破壞
- 字軌格式需符合：
  - 兩碼英文 + 八碼數字
- 發票期別須為六碼（YYYYMM）
- 匯入資料不可與系統既有字軌重複

---

## 六、常見問題（FAQ）

### Q：匯入出現格式錯誤怎麼辦？
A：請確認 CSV 欄位格式、期別與字軌代碼是否正確，避免使用 Excel 另存。

### Q：為何系統無法自動取得字軌？
A：多為尚未授權矽聯為加值中心，或未啟用相關模組。

### Q：匯入成功但無法開立發票？
A：請確認字軌是否已成功切本配號，並檢查剩餘號碼。可前往【發票字軌號碼管理】頁面，確認新匯入之字軌是否已列入有效本數並成功切本配號。

---

## 七、錯誤情境與處置方式

| 情境 | 建議處置 |
| --- | --- |
| CSV 匯入失敗（欄位格式錯誤） | 確認格式符合規範、發票期別與欄位名稱無誤 |
| 提示字軌重複 | 系統已存在相同字軌範圍，請確認是否重複上傳 |
| 開立發票時無字軌可用 | 請進入字軌管理檢查是否已完成切本配號作業 |
| 亂碼或檔案解析失敗 | 請改用 UTF-8 編碼的純文字 CSV 上傳，避免用 Excel 儲存造成破壞 |
