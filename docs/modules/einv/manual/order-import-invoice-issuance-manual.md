## 一、手冊目的

本手冊說明 **e首發票（EINV）後台「訂單匯入開立電子發票（訂單版）」的實際操作方式**，  
適用於 **未使用 API 或 FTP 串接方式** 的營業人，透過標準 Excel 範本匯入訂單資料，由系統自動完成電子發票開立。

> 本文件為 **非 SSOT 操作說明文件**。  
> 發票取號責任、時效與治理規則，請以規則文件為準。

---

## 二、適用對象

- 中小型企業
- 初期導入電子發票之營業人
- 未建置 API / FTP 串接者
- 客服人員（協助操作與錯誤排查）

---

## 三、功能說明

訂單匯入開立（訂單版）具備以下功能特性：

- 透過 **標準 Excel 範本** 批次匯入訂單
- 由 **e首發票系統自動取號與稽核**
- 系統自動判斷 **B2B / B2C** 發票（依統編）
- 支援：
  - Email 通知
  - 簡訊通知
  - 載具歸戶
- 提供完整發票流程：
  - 開立
  - 折讓
  - 作廢

> 單批建議不超過 **300 筆**；大量匯入請評估使用 FTP 模組。

---

## 四、操作步驟

### （一）下載 Excel 範本

後台路徑：

```
電子發票管理系統
＞ 銷項發票管理
＞ 線上操作手冊
＞ 相關巨集範本下載
```

請下載 **「訂單版」Excel 範本**。

---

### （二）填寫訂單資料

- 請依範本格式填寫，不可刪除或調整標頭列
- 常見欄位包含：
  - 訂單編號
  - 買方名稱／統編
  - 品名
  - 數量、單價
  - 小計、總金額
  - 稅別
  - 載具資訊（如有）

#### 稅別填寫說明（操作提醒）

| 稅別代碼 | 說明 |
|---|---|
| 1 | 應稅 |
| 2 | 零稅 |
| 3 | 免稅 |
| 9 | 混稅 |

---

### （三）匯入訂單並開立發票

後台路徑：

```
電子發票管理系統
＞ 銷項發票管理
＞ 訂單匯入開立（當日）發票作業
```

1. 上傳 Excel 檔案  
2. 點選【匯入 Excel 開立】

---

### （四）資料檢核與拋轉

- 系統會檢核：
  - 統編格式
  - 稅別
  - 手機載具格式
- 檢核完成後：
  - 點選【確定拋轉】
  - 系統完成發票開立
- 發票將依訂單資料：
  - 發送 Email / 簡訊
  - 或歸戶至會員載具

---

## 五、注意事項

- 同一 **訂單編號不可重複開立**
- 如需重開，請先作廢原發票
- 不得修改 Excel 範本格式或加入公式
- 單批匯入建議不超過 300 筆
- 發票需於規定時間內完成上傳：
  - **C 類：48 小時內**
  - **B 類：7 天內**
- 匯入完成後，請務必確認開立結果

## 六、常見問題（FAQ）

### Q：發票沒有開成功怎麼辦？
A：於匯入畫面系統會直接顯示錯誤訊息，請依提示修正後重新匯入。

### Q：統編填錯會怎樣？
A：系統會拒絕開立，請修正為合法 8 碼數字。

### Q：為什麼訂單只能開一次發票？
A：系統以訂單編號防止重複開立。

### Q：可以大量匯入嗎？
A：單批建議 300 筆內，超過請評估 FTP 模組。

## 七、錯誤情境與處置方式（操作層級）

### 圖示說明
- ⚠ 需留意，建議盡快處理  
- ✘ 必須修正，否則無法開立  

| 情境 | 處置方式 | 優先等級 |
|---|---|---|
| ⚠ 資料錯誤但已開立 | 作廢後重新匯入 | 高 |
| ⚠ 資料錯誤未開立 | 可直接離開畫面 | 中 |
| ✘ 統編錯誤 | 修正後重新匯入 | 高 |
| ✘ 稅別錯誤 | 補齊正確代碼 | 高 |
| ⚠ 重複訂單 | 作廢後重開 | 中 |
| ✘ 載具格式錯誤 | 修正格式後上傳 | 高 |
| ⚠ 系統逾時 / 網路異常 | 稍後重試或聯繫客服 | 中 |
