# EasyStore 自動開立電子發票操作手冊

## 一、手冊目的

本手冊說明營業人於 **EasyStore 平台** 上 **自動開立電子發票** 的完整操作流程，適用於訂單付款完成後由系統自動拋轉發票至 e首發票平台。

> 本文件為 **非 SSOT 操作說明文件**，僅說明系統操作方式。

---

## 二、功能說明

- 訂單付款完成後，自動由系統拋轉發票至 e首發票平台
- 減少人工操作，避免遺漏或錯誤
- 協助營業人快速完成 B2C 與 B2B 發票的日常開立需求

---

## 三、操作步驟

1. 登入 EasyStore 後台，進入 **擴充模組 > e首發票**
2. 確認已輸入公司統編與 SALT 並完成設定
3. 於 **稅金與商品設定** 中勾選正確稅別（應稅或免稅）
4. 當客戶下單並完成付款：
   - 系統會自動帶入訂單資訊並開立發票
   - 發票號碼將自動同步回寫至訂單明細中
5. 登入 e首發票後台，確認發票是否成功上傳至財政部平台

---

## 四、注意事項

- 僅支援 **TWD 幣別** 訂單自動開立發票
- B2B 發票不得為混稅，如訂單含應稅與免稅商品，系統僅會開立 B2C 發票，B2B 訂單需分單
- SALT 為固定驗證碼，一般情況下不需更新
- 自動開立需以訂單狀態「已付款」為前提，未付款訂單不會自動開立發票
- 系統顯示「開立成功」不代表已回傳成功，需確認 e首發票後台回傳狀態
- **高流量情境下可能延遲 5~10 分鐘才完成開立，請耐心等待或查詢後台狀態**

---

## 五、常見問題（FAQ）

| 問題 | 解答 |
| --- | --- |
| 為什麼付款後沒有自動開立發票？ | 請確認是否已完成統編與 SALT 設定，且訂單幣別為 TWD |
| 自動開立的發票會同步回訂單嗎？ | 會，發票號碼將自動回寫至訂單明細 |
| 若發票開立金額錯誤怎麼辦？ | 請作廢錯誤發票，並重新再次開立 |
| B2B 訂單混稅會怎樣？ | 系統不會開立發票，需分單後重新開立 |
| 系統顯示開立成功，但後台查不到發票？ | 請至 e首發票後台確認回傳狀態，若未成功需手動處理 |
| 自動開立會延遲嗎？ | 有時會延遲 5~10 分鐘，請稍後再查詢或於後台確認狀態 |
| 遇到重複開立怎麼辦？ | 請作廢多餘發票，並保留正確發票即可 |

---

## 六、錯誤情境與處置方式

| 錯誤情境 | 處置方式 |
| --- | --- |
| 付款後未自動開立 | 確認設定是否正確（統編、SALT、稅金設定、幣別） |
| 開立金額不符 | 作廢錯誤發票，再依正確金額重新再次開立 |
| B2B 訂單混稅 | 系統不會開立發票，需分單重新開立 |
| 開立狀態成功但未回傳財政部 | 請登入 e首發票後台確認回傳，若失敗需手動上傳或聯繫客服 |
| 重複開立 | 請立即作廢重複開立的發票，僅保留正確的一張 |
| 排程未觸發 | 請檢查自動排程設定或 API 服務狀態，必要時重新排程或通知技術支援 |

---

## 七、合規提醒

- **開立時限**：B2C 需於 48 小時內回傳至財政部平台；B2B 需於 7 日內回傳
- **禁止事項**：不得自動開立 B2B 混稅發票；不得開立金額為零或負數之發票；不得重複開立發票
- **保存要求**：自動開立發票資料需保存至少 5 年，備稅務查核
