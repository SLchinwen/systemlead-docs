# 蝦皮 API 賣場授權操作手冊

## 一、手冊目的

本手冊說明 **e首發票（EINV）系統中「蝦皮 API 賣場授權」的實際操作方式**，讓系統可自動讀取蝦皮訂單與退貨資訊，並依設定規則自動開立電子發票或折讓單，確保發票符合財稅規範，降低漏開、短開或跨期開立的稅務風險。

> 本文件為 **非 SSOT 操作說明文件**，僅說明系統操作方式。  
> 授權範圍、時限與財稅責任，請以相關規則與合約為準。

---

## 二、功能說明

蝦皮 API 賣場授權完成後，系統可自動完成以下作業：

- 自動讀取蝦皮訂單
- 系統排程 **48 小時內** 自動開立電子發票
- 每 **7 天** 自動讀取退貨訂單並自動開立折讓單
- 自動同步訂單狀態，減少人工操作風險

---

## 三、操作路徑

```
電子發票管理系統
＞ EC 外部訂單 API 整合
＞ 蝦皮訂單讀取授權作業
```

---

## 四、操作步驟

### （一）進入授權畫面

依上述路徑進入【蝦皮訂單讀取授權作業】。

### （二）新增授權

1. 點擊【新增授權】
2. 選擇賣場地區：
   - 台灣賣場 → **TW**
   - 中國賣場 → **Shopee**
3. 輸入蝦皮 **賣場登入驗證碼**
4. 確認授權資訊無誤後，點擊【確認】
5. 選擇 **授權 365 天**，並按下【同意】
6. 點擊橘色【變更】按鈕，將接收狀態固定為：
   - **準備出貨 (READ_TO_SHIP)**

## 五、狀態設定

- **固定狀態**：準備出貨 (READ_TO_SHIP)

> 依財稅規範，發票須於「收款或出貨時」開立。系統已固定為「準備出貨」狀態，以避免跨期、漏開或短開發票。若營業人希望改回「出貨中 (SHIPPED)」，需自行承擔稅務風險並額外申請修改。

---

## 六、注意事項

- 授權期限建議設定為 **365 天**，確保不中斷
- 系統會於 **訂單進入準備出貨後 48 小時** 自動排程開立發票
- 授權完成後，系統每 **7 天** 會自動讀取蝦皮退貨訂單，並自動開立折讓單
- 系統上線初期，可能無法接收授權前的訂單狀態，需人工確認是否需補開發票
- 建議使用 **Chrome 瀏覽器** 進行授權，以避免相容性問題

## 七、常見問題（FAQ）

### Q：授權需要多久時間生效？
A：一般授權確認後即可立即生效，無需等待。

### Q：授權會不會過期？
A：預設授權期限為 365 天，到期後需重新授權。

### Q：訂單狀態是否可以自行更改？
A：目前狀態固定為「準備出貨 (READ_TO_SHIP)」，不可修改。

### Q：授權後能否重設？
A：如需重新授權，需重新輸入驗證碼並進行設定，但訂單狀態維持固定為「準備出貨」。

### Q：如何同時授權多個蝦皮賣場？
A：每個賣場需分別進行授權，並在系統內確認授權狀態。

### Q：跨境店舖（TW / Shopee）需要分開授權嗎？
A：是的，不同地區的賣場需分開進行授權，避免資料混淆。

### Q：如果訂單在授權前就成立，會自動開立嗎？
A：不會，需人工補開，系統僅處理授權完成後的新訂單。

## 八、錯誤情境與處置方式

| 錯誤情境 | 建議處置 |
| --- | --- |
| 授權驗證碼輸入錯誤 | 重新取得蝦皮提供的驗證碼後再嘗試 |
| 授權未成功，系統無法讀取訂單 | 請確認賣場選擇是否正確（TW / Shopee），並重新授權 |
| 授權過期 | 重新進行授權，建議設定 365 天有效期 |
| 網路中斷或蝦皮伺服器維護 | 建議先人工補開發票，待恢復後再比對同步 |
| 訂單已開立發票但未回傳蝦皮 | 需人工比對並重新同步發票狀態 |
