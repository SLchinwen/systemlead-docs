# EasyStore 再次開立電子發票操作手冊

## 一、手冊目的

本手冊說明營業人於 **EasyStore 平台** 上 **再次開立電子發票** 的完整操作步驟，適用於原發票因錯誤或作廢後需重新再次開立，或訂單交易正常但因系統或人為原因未能成功開立發票時。

> 本文件為 **非 SSOT 操作說明文件**，僅說明系統操作方式。

---

## 二、功能說明與禁止使用情境

**適用情境**：
- 原發票因錯誤或作廢後需重新再次開立
- 訂單交易正常，但因系統或人為原因未能成功開立發票

**禁止使用情境**：
- 訂單已存在有效發票，不得重複再次開立
- 跨期再次開立需先開立折讓單後，再重新開立發票
- 不得再次開立零金額或不符規定的發票

---

## 三、操作步驟

1. 登入 EasyStore 後台，進入 **訂單管理 > 所有訂單 > 點選訂單編號 > 訂單明細 > e首發票電子發票系統**
2. 確認原發票狀態（如已作廢）
3. 點擊【再次開立】按鈕，系統將自動帶入訂單資訊
4. 確認或修改客戶統編、課稅別及發票金額
5. 按下【確認送出】，系統將回傳新發票號碼並同步至 e首發票後台
- **再次開立僅在財政部平台回傳「成功」狀態後，才算正式生效**

---

## 四、跨期再次開立流程說明

- 當發票已作廢，可於 EasyStore 後台直接點擊【再次開立】，系統將帶入訂單資訊完成新發票開立
- **若發票已跨期**，無法於 EasyStore 平台直接再次開立：
  - 需於 e首發票後台先開立折讓單
  - 然後透過 Excel 檔重新匯入訂單資料至 e首發票系統，手動開立發票

---

## 五、注意事項

- 再次開立需符合原訂單交易實際金額，不得任意更改
- 跨期再次開立需依規定：必須先開立折讓單後，再重新開立發票
- 再次開立完成後，請於 e首發票後台確認是否已回傳成功
- 若因作廢而再次開立，請保留原發票與折讓單紀錄，以備查核

---

## 六、常見問題（FAQ）

| 問題 | 解答 |
| --- | --- |
| 什麼情況下需要再次開立發票？ | 原發票作廢或系統未成功開立時 |
| 再次開立與重新開立有何不同？ | 兩者意思相同，皆指針對訂單重新開立發票 |
| 跨期訂單可以再次開立嗎？ | 不可直接再次開立，需先開立折讓單，再重新開立發票 |
| 再次開立後會有新發票號碼嗎？ | 會，系統會重新分配新的發票號碼 |
| 如果再次開立失敗怎麼辦？ | 請確認發票號碼是否足夠，並查詢後台是否正常回傳，必要時聯繫客服 |
| 再次開立後多久能查到結果？ | 一般需 5~10 分鐘，請至 e首發票後台確認，或隔日再次查詢 |
| 如果誤按再次開立怎麼辦？ | 請立即作廢該發票，並依正確資訊重新開立 |

---

## 七、錯誤情境與處置方式

| 錯誤情境 | 處置方式 |
| --- | --- |
| 訂單已有有效發票 | 不得再次開立，避免重複開立 |
| 跨期訂單再次開立 | 需先開立折讓單，再重新開立發票 |
| 再次開立失敗（號碼不足） | 登入財政部電子發票整合服務平台確認是否有未取號本數，若已用完需向國稅局申請新增本數 |
| 再次開立發票資訊錯誤 | 作廢錯誤發票，依正確資訊重新再次開立 |
| 誤按再次開立 | 請立即作廢該發票，並依正確資訊重新操作 |

---

## 八、合規提醒

- **再次開立時限**：應於交易發生日當期完成，跨期需依規定以折讓方式處理
- **禁止事項**：不得對已有有效發票的訂單再次開立；不得再次開立零金額或不符規定之發票；不得開立或再次開立 B2B 混稅發票
- **保存要求**：再次開立發票及相關折讓紀錄需保存至少 5 年，備稅務查核
