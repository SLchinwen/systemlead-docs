# EasyStore 手動開立電子發票操作手冊

## 一、手冊目的

本手冊說明營業人於 **EasyStore 平台** 上 **手動開立電子發票** 的完整操作步驟，適用於自動開立未成功時的補救，或特殊情境需人工確認或修正發票內容時。

> 本文件為 **非 SSOT 操作說明文件**，僅說明系統操作方式。

---

## 二、功能說明與禁止使用情境

**適用情境**：
- 自動開立未成功時的補救措施
- 特殊情境需人工確認或修正發票內容時

**禁止使用情境**：
- 訂單已自動開立成功，不可再次手動開立
- 訂單尚未付款時，不可提前手動開立
- 不得重複對同一訂單號碼開立發票
- 不得手動開立跨期發票，跨期應以折讓單或報備國稅局方式處理

---

## 三、操作步驟

### （一）進入訂單明細

- 登入 EasyStore 後台
- 路徑：**訂單管理 > 所有訂單 > 點選訂單編號 > 訂單明細 > e首發票電子發票系統**

### （二）確認客戶及發票資訊

確認以下欄位是否正確：
- **訂單編號**（不可修改，避免重複開立）
- **課稅別**（可選擇應稅或免稅，不支援零稅率）
- **發票金額**（可依實際調整，異動後需再次確認）
- **商品資料**（可手動修改，異動後需再次確認）

### （三）送出開立發票

- 確認資訊正確後，按下【送出】完成開立
- 系統將同步將發票號碼傳送至 e首發票後台，可於「電子發票」頁面查詢

---

## 四、注意事項

- 手動開立功能僅適用於 **當期發票**，跨期發票需報備國稅局或改以折讓處理
- 訂單狀態若尚未「已付款」，請勿提前開立，以避免發票錯誤
- 請確認課稅別與商品設定正確，避免 B2B 混稅發票無法開立
- 開立前請先確認是否已有自動開立紀錄，避免重複開立
- 建議於每日開立後，至 e首發票後台查詢確認上傳狀態

---

## 五、常見問題（FAQ）

| 問題 | 解答 |
| --- | --- |
| 手動開立發票後未成功 | 檢查是否為跨期發票，或確認是否有合法字軌可用 |
| 金額與訂單不符怎麼辦？ | 可手動修改金額，但需再次確認避免開立異常 |
| 為什麼零稅率發票無法開立？ | 系統不支援手動開立零稅率發票，需透過其他方式處理 |
| B2B 混稅訂單無法開立 | 依法規規定，B2B 不得開立混稅發票，請消費者分單 |
| 遇到「401 Unauthorized」錯誤？ | 檢查 SALT 與統編授權是否正確，並確認帳號是否具管理員權限 |
| 遇到「500 系統錯誤」？ | 系統異常，請稍後再試，若持續發生請聯繫客服並提供訂單編號與時間 |
| 發票號碼不足怎麼辦？ | 先登入財政部電子發票整合服務平台確認是否有未取號本數，若字軌已取用完畢，需向國稅局申請新增本數 |

---

## 六、錯誤情境與處置方式

| 錯誤情境 | 處置方式 |
| --- | --- |
| 訂單未帶入統編或課稅別 | 作廢原發票後，補齊客戶資訊並重新開立 |
| 異動金額後仍無法開立 | 確認是否超出原始發票金額，避免違反規範 |
| 發票狀態顯示「失敗」 | 登入 e首發票後台查詢，必要時聯繫客服並提供訂單編號追蹤 |
| 跨期發票需作廢或折讓 | 當期可作廢，跨期需報備國稅局並以折讓單處理 |
| 發票號碼不足 | 先登入財政部電子發票整合服務平台確認未取號本數，若已用完需向國稅局申請新增本數 |
| 訂單重複開立 | 請先作廢原發票，再重新開立，避免違反稅法 |

---

## 七、合規提醒

- **開立時限**：B2C 開立後需於 48 小時內上傳；B2B 需於 7 日內上傳至財政部平台
- **禁止事項**：不得開立 B2B 混稅發票；不得開立金額為零或負數之發票；不得重複開立同一訂單發票
- **保存要求**：電子發票與相關資料須保存至少 5 年，備稅務查核
