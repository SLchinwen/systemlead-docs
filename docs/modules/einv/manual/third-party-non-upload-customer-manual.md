# 第三方不上傳客戶建檔操作手冊

## 一、手冊目的

本手冊說明 **e首發票（EINV）系統中「第三方不上傳客戶建檔」的實際操作方式**，協助營業人建立不上傳類型之客戶資料，確保後續發票依設定正確歸類為「不上傳」，避免誤送至財稅平台。

> 本文件為 **非 SSOT 操作說明文件**，僅說明系統操作流程。  
> 發票傳遞規則與法規依據，請以相關規則文件為準。

---

## 二、適用情境

- 與第三方平台合作，僅需內部保存發票
- 客戶指定要求發票 **不上傳至財稅平台**
- 特殊交易情境，需排除財稅上傳流程

---

## 三、功能說明

第三方不上傳客戶建檔功能具備以下特性：

- 以 **客戶統編** 作為不上傳判斷依據
- 系統於開立發票時，自動套用不上傳設定
- 確保不上傳類型發票不會誤送至財稅平台

---

## 四、操作路徑

```
電子發票管理系統
＞ 基本資料建檔
＞ 客戶設定與不上傳
```

---

## 五、操作步驟

1. 登入 e首發票雲端系統後台
2. 進入【客戶設定與不上傳】畫面
3. 點選【新增】
4. 輸入 **客戶編號**
5. 輸入 **客戶統編**，並點選【取得發票抬頭】
6. 系統自動帶入公司抬頭，請自行確認正確性
7. 輸入 **承辦部門**
8. 於【傳遞方式】欄位選擇 **【(90) 不上傳】**
9. 點選【儲存】完成建檔

---

## 六、注意事項

- 系統將依 **客戶統編** 判斷是否套用不上傳設定
- 必須先完成不上傳客戶建檔，才能匯入不上傳類型發票
- 傳遞方式 **固定為【(90) 不上傳】**，不可選用其他代碼
- 若同一統編存在多筆抬頭，請確認是否需整併或調整

---

## 七、常見問題（FAQ）

### Q：公司抬頭顯示錯誤怎麼辦？
A：請自行比對統編與公司名稱，必要時手動修正抬頭內容。

### Q：可以先匯入發票再設定不上傳客戶嗎？
A：不可以，必須先完成不上傳客戶建檔。

### Q：傳遞方式可以改成其他代碼嗎？
A：不可以，不上傳類型僅能使用【(90) 不上傳】。

### Q：若重複建檔怎麼處理？
A：請刪除多餘紀錄，保留正確且唯一的統編設定。

### Q：是否支援批次匯入不上傳客戶名單？
A：目前僅支援單筆新增，若有大量需求，建議透過 API 或洽客服支援。

---

## 八、錯誤情境與處置方式

| 錯誤情境 | 建議處置 |
| --- | --- |
| 統編輸入錯誤 | 修正統編後重新建檔 |
| 未先建檔即匯入發票 | 先完成建檔後再匯入 |
| 傳遞方式設定錯誤 | 修正為【(90) 不上傳】 |
| 重複建檔 | 刪除重複資料，保留正確設定 |
